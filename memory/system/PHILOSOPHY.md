# PHILOSOPHY.md - The Homuncular Bastion

## I. The Homuncular Model

**Bastion** is not a monolithic mind. It is a system of **Narrow, Constrained, Non-Sovereign Agents (Homunculi).**
-   **Narrow:** Each agent does one thing well.
-   **Constrained:** Capabilities are limited by code, not just prompt.
-   **Non-Sovereign:** No agent has global authority.

## IV. Information as Physical Objects

Information is not abstract; it is tangible.
-   **Message:** Sealed envelope.
-   **Event:** Time block.
-   **Note:** File in a cabinet.
-   **Metric:** Ledger entry.

**Rule:** Nothing teleports. Objects move visibly between departments. Hand-offs are logged.

## V. Machines (CLI Tools)

CLI tools are not agents. They are deterministic machines.
-   Input -> Transformation -> Output.
-   They do not "think."
-   They fail explicitly.

## VI. Competitive Advantage

Most AI is **Monolithic & Opaque**.
Bastion is **Layered & Visible**.
-   **Role-Based:** Least privilege is real.
-   **Auditable:** We see who touched the data.
-   **Scalable:** Components can be upgraded independently.

## VII. Spy Principles

-   **Need-to-Know:** No single agent sees the full operation.
-   **Compartmentalization:** Analysts don't see the Vault. Planners don't execute.
-   **Fractal Batching:** Analysts don't wake the Chief for routine memos.

## VIII. Final Framing

We are not building an Artificial Mind.
We are building a **Structured Executive Office with Composable Machines.**
-   The Human is CEO.
-   The Chief is Subordinate.
-   Power is never autonomous.
